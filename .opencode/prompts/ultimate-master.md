# ULTIMATE MASTER PROMPT --- RYO Engineering Framework

You are the lead engineer responsible for transforming this repository
into a production-grade CLI framework.

## Mission

Build, refactor, polish, document, and validate the entire project until
it is production-ready.

## Core Rules

-   Never reduce existing functionality.
-   Preserve passing tests.
-   Prefer small, atomic commits.
-   Use strict TypeScript.
-   Remove dead code.
-   Refactor duplicated logic.
-   Keep architecture clean.

## Development Phases

### Phase 1 --- Full Audit

-   Audit the entire repository.
-   Detect bugs, code smells, architecture issues.
-   Produce an internal execution plan.

### Phase 2 --- Core Architecture

-   Refactor command system.
-   Improve registry loader.
-   Improve skill loader.
-   Improve filesystem utilities.
-   Improve logger.

### Phase 3 --- CLI UX

-   Polish help output.
-   Better error messages.
-   Better success messages.
-   Add friendly empty states.

### Phase 4 --- Branding

Display a startup banner:

RYO

Requirements: - Only the word "RYO" uses pink. - Clean minimal design. -
Similar polish to modern CLIs such as Gemini CLI/OpenCode. - Version
displayed below.

### Phase 5 --- Skills

Maintain and improve existing skills. Add official skill templates
for: - Laravel - React - Next.js - Node.js - Express - TypeScript -
Docker - Kubernetes - Git - PostgreSQL - MySQL - Redis - Linux -
Python - FastAPI - Java Spring Boot - C# - Go - Flutter - React Native -
Machine Learning - Data Science - Competitive Programming - Clean
Architecture - System Design - Microservices - CI/CD - DevOps -
Testing - REST API - GraphQL

Each skill must include: - metadata - manifest - README - VERSION -
CHANGELOG - prompts - references - troubleshooting - checklist -
examples

### Phase 6 --- Testing

Increase coverage. Fix flaky tests. Ensure deterministic integration
tests.

### Phase 7 --- Performance

Optimize startup. Reduce unnecessary filesystem access. Improve loading
speed.

### Phase 8 --- Documentation

Update README. Document commands. Document architecture. Document
contribution guide.

### Phase 9 --- Release

Validate build. Validate lint. Validate tests. Ensure cross-platform
compatibility.

### Final Validation Checklist

-   Build passes
-   Tests pass
-   Lint passes
-   No TODO/FIXME
-   No duplicated code
-   Clean architecture
-   Production ready

Repeat improvements until no meaningful issues remain.
