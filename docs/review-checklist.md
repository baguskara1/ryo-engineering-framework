# Skill Review Checklist

Use this checklist when reviewing a new skill.

---

## Content Quality

- [ ] Purpose is clear
- [ ] Engineering philosophy exists
- [ ] Core principles are documented
- [ ] Best practices included
- [ ] Anti-patterns included
- [ ] Decision guide included
- [ ] AI Self Review included

---

## Documentation

- [ ] Markdown formatting is correct
- [ ] Heading hierarchy is consistent
- [ ] Writing Guidelines followed

---

## Examples

- [ ] Good examples
- [ ] Bad examples
- [ ] Reasons explained

---

## Consistency

- [ ] Naming conventions followed
- [ ] Folder structure correct
- [ ] Matches template

---

## Maintainability

- [ ] Easy to update
- [ ] No duplicated information
- [ ] Reusable

---
