# Style Guide

## Markdown

Use ATX headings.

Keep line length readable.

Prefer examples.

## Folder Naming

Use lowercase.

Use kebab-case.

Example

nextjs

clean-code

code-review

## Skill Naming

One skill per folder.

Every skill should eventually contain

- manifest.yaml
- SKILL.md
- examples.md
- checklist.md
- CHANGELOG.md
