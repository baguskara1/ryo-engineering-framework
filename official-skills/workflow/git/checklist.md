# Checklist

- [ ] Commit messages follow Conventional Commits format
- [ ] Branch names follow team convention (e.g., `feature/`, `fix/`, `chore/`)
- [ ] No large binary files committed (use `.gitignore` and `git-lfs`)
- [ ] Rebasing used to maintain clean history before merging
- [ ] Pull requests reviewed before merging to main branch
- [ ] Signed commits enabled for security
- [ ] `.gitignore` configured for the project language/framework
- [ ] Git hooks (pre-commit, commit-msg) configured for linting
- [ ] Remote tracking branches synchronized regularly
- [ ] Tags used for releases following semver
