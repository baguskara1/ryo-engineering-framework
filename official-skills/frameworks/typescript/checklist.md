# Checklist

- [ ] `strict: true` enabled in tsconfig.json
- [ ] Explicit return types on functions and methods
- [ ] Interfaces preferred over type aliases for object shapes
- [ ] Union and intersection types used appropriately
- [ ] Generics used for reusable, type-safe utilities
- [ ] `unknown` preferred over `any` for type-safe dynamic values
- [ ] Type guards or assertion functions used for type narrowing
- [ ] `@types/*` packages installed for third-party libraries
- [ ] ESLint + `typescript-eslint` configured for linting
- [ ] Build output configured with proper module format and target
