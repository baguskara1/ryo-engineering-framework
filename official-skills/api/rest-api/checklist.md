# Checklist

- [ ] Resources named as nouns (plural), not verbs
- [ ] HTTP methods used correctly (GET, POST, PUT, PATCH, DELETE)
- [ ] Consistent URL structure with proper nesting for relationships
- [ ] Meaningful HTTP status codes for all responses
- [ ] Input validation with clear error messages
- [ ] Pagination implemented for list endpoints
- [ ] Authentication and authorization enforced
- [ ] API versioning strategy defined
- [ ] OpenAPI/Swagger documentation maintained
- [ ] Rate limiting and throttling configured
