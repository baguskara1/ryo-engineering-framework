# Checklist

- [ ] Schema designed with clear types, inputs, and enums
- [ ] Queries for reading data, mutations for writing
- [ ] Resolvers follow single responsibility principle
- [ ] DataLoader used to batch database queries
- [ ] Input validation on mutations with custom scalars
- [ ] Authentication and authorization enforced in resolvers
- [ ] Query depth limiting and complexity analysis configured
- [ ] Subscriptions use proper pub/sub mechanism
- [ ] Error handling with standardized error format
- [ ] Schema documentation with descriptions on types/fields
