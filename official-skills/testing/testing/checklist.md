# Checklist

- [ ] Unit tests cover business logic and edge cases
- [ ] Integration tests verify external service interactions
- [ ] Test fixtures and factories used for consistent test data
- [ ] Mocking limited to external boundaries (IO, network)
- [ ] Tests are independent and can run in any order
- [ ] CI pipeline runs tests on every commit
- [ ] Code coverage thresholds defined and enforced
- [ ] E2E tests cover critical user flows
- [ ] Performance/load tests written for critical endpoints
- [ ] Test documentation describes how to run and write tests
