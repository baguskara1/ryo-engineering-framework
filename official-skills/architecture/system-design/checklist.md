# Checklist

- [ ] Requirements clarified (functional and non-functional)
- [ ] Capacity estimation calculated (QPS, storage, bandwidth)
- [ ] Data model designed with appropriate storage technology
- [ ] API design follows REST or GraphQL conventions
- [ ] Caching strategy defined (CDN, application cache, database cache)
- [ ] Load balancing and auto-scaling configured
- [ ] Database replication and sharding strategy defined
- [ ] Monitoring and alerting planned
- [ ] Disaster recovery and backup strategy documented
- [ ] Trade-offs documented (consistency vs availability, latency vs throughput)
