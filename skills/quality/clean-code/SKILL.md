# Clean Code Engineering Skill

## Identity

Clean code is code that is easy to understand, modify, test, and maintain.

---

# Engineering Philosophy

- Optimize for readability.
- Prefer simplicity.
- Reduce cognitive load.
- Design for change.

---

# Core Principles

- Meaningful naming
- Single Responsibility Principle
- Separation of Concerns
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple)
- YAGNI (You Aren't Gonna Need It)

---

# Anti Patterns

- God Objects
- Long Functions
- Magic Numbers
- Deep Nesting
- Duplicate Code

---

# AI Self Review

- [ ] Easy to understand
- [ ] Easy to test
- [ ] Easy to maintain
- [ ] Minimal complexity
- [ ] No unnecessary duplication

---

# Summary

Write code for humans first. Computers execute code once; developers read it many times.
