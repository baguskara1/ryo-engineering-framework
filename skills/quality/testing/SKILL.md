# Testing Engineering Skill

## Identity

Testing verifies that software behaves correctly and continues to work as the code evolves.

---

# Engineering Philosophy

- Test behavior, not implementation.
- Keep tests fast and deterministic.
- Prefer confidence over coverage percentage.
- Tests should document expected behavior.

---

# Test Pyramid

1. Unit Tests
2. Integration Tests
3. End-to-End Tests

Favor more unit tests than end-to-end tests.

---

# Best Practices

- Write descriptive test names.
- Use Arrange-Act-Assert.
- Test edge cases.
- Test failure scenarios.
- Keep tests independent.

---

# Anti Patterns

- Testing implementation details.
- Excessive mocking.
- Flaky tests.
- Shared mutable state.
- Copy-paste test cases.

---

# AI Self Review

- [ ] Critical logic tested
- [ ] Edge cases covered
- [ ] Tests are readable
- [ ] Tests are deterministic
- [ ] Easy to maintain

---

# Summary

Good tests increase confidence, enable safe refactoring, and serve as living documentation.
